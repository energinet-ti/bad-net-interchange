# CGMA vs IGM Comparator

A modern, high-performance tool for comparing 2D SSH scenarios from IGM and CGMA inhouse net positions. This project features a browser-based web app powered by Rust and WebAssembly (WASM) for fast, client-side XML parsing and comparison.

---

## 🚀 Live Demo

The latest version is automatically deployed to GitHub Pages:  
**[energinet-ti.github.io/bad-net-interchange](https://energinet-ti.github.io/bad-net-interchange/)**

---

## 🗂️ Project Structure

- **sample-data/**: Example IGM and CGMA files for testing.
- **src/rust-wasm/**: Rust source code compiled to WebAssembly for high-performance XML parsing and comparison logic.
- **src/web/**: Source for the web frontend (HTML, JS, CSS).
- **docs/**: Built static web app, ready to deploy or open locally.
- **docs/pkg/**: WASM and JS glue code generated by the Rust build.

---

## ⚙️ How It Works

- **Rust code** in `src/rust-wasm` is compiled to WASM using `wasm-pack`. This enables efficient XML parsing and comparison directly in the browser.
- **Web frontend** (`src/web`) provides a user-friendly UI for uploading and comparing files, loading the WASM module for processing.
- **Build script** (`src/build_wasm_webapp.sh`) automates the build process, copying all necessary files to `docs/` for deployment.

---

## 🛠️ Building Locally

1. **Install Rust and wasm-pack**  
	- [Install Rust](https://rustup.rs)  
	- Install wasm-pack:  
	  ```
	  cargo install wasm-pack
	  ```

2. **Build the WASM web app**  
	From the project root:
	```
	bash src/build_wasm_webapp.sh
	```
	This will:
	- Compile Rust to WASM and output to `docs/pkg/`
	- Copy web assets to `docs/`
	- The static site is now ready in `docs/`

3. **Open the app**  
	Open `docs/index.html` in your browser.

## 🦀 Rust/WASM Quick Guide

- Rust code is in `src/rust-wasm/`.
- Dependencies: `wasm-bindgen`, `serde`, `roxmltree`, `zip`.
- Build for web:  
  ```
  wasm-pack build --target web --release
  ```
- The generated WASM and JS glue code are loaded by the frontend.

**WebAssembly (WASM)** enables running Rust code in the browser for performance-critical tasks, such as parsing large XML files.

---

## 💡 Development Tips

- Edit Rust code in `src/rust-wasm/`, then rebuild with the script.
- Edit frontend code in `src/web/`, then rebuild to update `docs/`.
- For new dependencies, update `Cargo.toml` and rebuild.
